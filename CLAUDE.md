# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

**Tutorly** is an Electron desktop app: an AI tutor that writes a course one lesson at a time, on the fly. The user types what they want to learn; the app infers a *mission*, generates the first lesson, and streams follow-up lessons/quizzes/projects on request. Lessons are generated by Claude against a JSON schema, so the model output is structured data (not free text) that the UI renders.

## Commands

```bash
npm run dev      # Vite dev server (127.0.0.1:5173) + Electron with hot reload
npm run build    # Vite build Рєњ dist/
npm start         # build, then launch Electron against dist/
```

There is **no test suite, linter, or formatter** configured. Don't assume `npm test`/`npm run lint` exist.

`dev` uses `concurrently`: it starts Vite, waits for the port, then launches Electron with `VITE_DEV_SERVER_URL` set. `main.cjs` loads that URL in dev and `dist/index.html` in production.

## Architecture

Two processes, bridged by a small contextIsolation API. Renderer never touches the network or filesystem directly.

- **`electron/main.cjs`** Рђћ main process. Owns the window, persistence, and all IPC handlers. Persists two JSON files under `app.getPath('userData')` (`~/Library/Application Support/Tutorly/` on macOS): `tutorly-data.json` (courses/lessons/threads/quiz state) and `tutorly-settings.json` (connection method, model, and an API key encrypted via Electron `safeStorage`). IPC handlers wrap the agent and return `{ ok, result }` / `{ ok, error }`.
- **`electron/preload.cjs`** Рђћ exposes `window.teach` with `loadState`, `saveState`, `connect`, `disconnect`, `setMethod`, `setModel`, `startCourse`, `nextLesson`, `askThread`. This is the only surface the renderer can call.
- **`electron/agent.cjs`** Рђћ the LLM backend. Three interchangeable methods selected by settings:
  - `'cli'` Рєњ `@anthropic-ai/claude-agent-sdk`, driving the local Claude Code sign-in (subscription auth, **no API key**). ESM-only, so it's `import()`-ed lazily from this CJS module.
  - `'api'` Рєњ `@anthropic-ai/sdk` with an API key (or env `ANTHROPIC_API_KEY` / `ant auth login`).
  - `'local'` Рєњ any OpenAI-compatible server at `cfg.baseUrl` (Ollama default `http://localhost:11434/v1`, LM Studio, llama.cpp, Рђд) via plain `fetch` to `/chat/completions`. No SDK; JSON is requested with `response_format: json_object` and parsed leniently (`parseJsonLoose`) since local models don't reliably honor a schema.
  All paths funnel through `generateJson()`. `listModels()` returns the selectable models per backend (live `GET /v1/models` for local, `models.list()` for api, static list for cli). `errMessage()` maps low-level errors to actionable user-facing strings Рђћ extend it rather than surfacing raw errors.
- **`electron/teach-prompts.cjs`** Рђћ the teaching brain: `TEACH_SYSTEM`/`THREAD_SYSTEM` system prompts and the JSON schemas (`COURSE_START_SCHEMA`, `NEXT_LESSON_SCHEMA`, `LESSON_SCHEMA`). A lesson's `type` is `lesson | quiz | project`; each lesson carries a `learningRecord` (1РђЊ3 sentences of the key insight) that is fed back into `courseContext()` so future lessons target the user's *zone of proximal development*. **Changing lesson shape means changing both the schema here and the renderer that reads it.**
- **`src/App.jsx`** Рђћ the entire UI, one ~1200-line React class component (screens are conditional renders off `this.state`, not routes). Owns all app state; calls `window.teach` (aliased as `api`); debounces saves 800ms via `persist()`. Lesson `bodyMarkdown` is rendered with `marked` then sanitized with `DOMPurify` before `dangerouslySetInnerHTML`. Supports selecting a passage to open an inline Q&A *thread* against that lesson.
- **`src/ui.jsx`** Рђћ small presentational helpers (`H` hover-aware element, inline SVG icons). `src/globals.css` + inline styles; no CSS framework.

### Data flow

`App` state Рєњ debounced `api.saveState()` Рєњ `state:save` IPC Рєњ `tutorly-data.json`. On mount, `state:load` hydrates courses/threads/quiz. Generation: UI calls `startCourse`/`nextLesson`/`askThread` Рєњ IPC Рєњ `agent.cjs` Рєњ Claude Рєњ schema-validated JSON Рєњ merged into state Рєњ persisted.

## Teaching methodology

The pedagogy (mission-grounding, storage vs. fluency strength, desirable difficulty, retrieval practice, quiz-option formatting rules) is adapted from Matt Pocock's `/teach` skill, vendored under `skills/teach/`. `skills/teach/SKILL.md` is the original filesystem-workspace version; `electron/teach-prompts.cjs` is the app's adaptation of the same philosophy. When adjusting how lessons are written, edit the system prompts in `teach-prompts.cjs` Рђћ `skills/teach/` is reference, not runtime.

## Conventions

- `electron/*` is CommonJS (`.cjs`, `require`); `src/*` is ESM + JSX bundled by Vite.
- Screenshot/automation hooks in `main.cjs` are gated on env vars `TEACH_SCREENSHOT`, `TEACH_EXEC`, `TEACH_EXEC_WAIT` Рђћ used to capture the window headlessly.
- Default model is `claude-opus-4-8` (`DEFAULT_MODEL` in `agent.cjs`); the picker list lives in `MODELS` in `App.jsx`. Keep the two in sync.
